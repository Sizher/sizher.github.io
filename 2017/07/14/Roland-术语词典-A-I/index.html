Roland 术语词典 (A-I)

A／D

即类比讯号 (Analog) 转换成数位讯号 (Digital)，例如将麦克风接上数位录音座进行收音时，麦克风所传送过来的讯号是类比讯号，输入到数位录音座时会经过一个 A/D 转换器，以将类比讯号转成数位格式，让数位录音座加以储存或做进一步处理。 反过来则是 D/A，即数位转成类比。

Algorithm「 演算法 」

这本来是学术 (如数学、程式) 领域中的用语，然而当套用在音乐电子器材上的时候，它指的是：不同的数位效果器串接的顺序。大部份的数位效果处理机都已含有各种演算法。

综合效果器的基本原理，就是内建各种单一效果 (如同吉他用的单颗效果器)，再将这些单一效果加以排列组合，如同您将几个 BOSS 单颗效果器以各种方式串接一样，如此会制作出各式各样的效果；而一个演算法，就是一种串接顺序。通常一个演算法当中的效果串接顺序是无法修改的，但像是 BOSS 的 VF-1 综合效果器，您可以改变演算法中某些效果的顺序；而 BOSS 的 GT-6 综合效果器，更没有所谓的预设演算法，您可以任意编排各种效果的顺序。

Analog「 类比 」

这是与「Digital」(数位) 相对的观念。

以声音的讯号为例，所谓的类比声音讯号 (Analog Audio)，是将声波化成实际的电压变化，由讯号导线传送或储存在磁带装置。如果用示波器去实际侦测类比讯号传输中的电压变化，则会在示波器上看到类似声波的图形显示。

相对的，数位声音讯号 (Digital Audio) 则是进一步将声波转化成数字，将这些数字进一步解码之后，变成只以 0 与 1 所表示的一串讯号。在实际的讯号传输过程中，就是以高电位来代表 1，以低电位代表 0 。若以示波器实际侦测数位讯号传输中的电压变化，则会看到类似下图的显示。

理论上，类比讯号就是将声波实际上的振动，完全转换成电位高低的变化，所以照理来说，类比讯号应能传达最完整的声音 —— 但事实上，由于讯号线材、输出入端子的物理特性，使得讯号会受到外界如磁场干扰、导体电阻等等的影响，最后到达的讯号多少会与原始讯号有所不同。 而数位讯号则是将类比的讯号加以分割，将每一个分割处上的振幅值转换成数字，换句话说，分割得越细，数位讯号就能呈现更逼真的声音，但是再怎么分割，也没有办法做到与类比讯号一模一样。

一般而言，若是将类比讯号每一秒分割成 44100 个点来转化成数位讯号，人耳大致上就分不出与原音的差别，这也就是所谓 CD 音质是 44.1kHz 取样频率的由来。 数位讯号的好处是：传输过程中不容易失真。这是因为数位讯号只有单纯的 0 与 1 两种，也就是低电位与高电位，在一般程度的干扰下，低电位几乎仍会保持在低电位的标准以下，高电位亦保持在高电位的标准以上；换句话说，代表 0 的讯号送达目标时，不太可能就变成 1 ，而代表 1 的讯号也不太可能变成 0 。相对的，类比讯号流中任一点的电位稍为受到干扰，就会影响到那一点的声波表现，这也是为什麽近来的音响、录音器材，都要往数位化发展的重要原因之一。

数位讯号的另一个好处是，容易编辑并储存。数位讯号几乎可以储存在现今的任何一种数位储存媒体上，另外，也比类比讯号更容易进行进一步的处理，如效果器、混音等等，使得数位录音已成为现代录音工业的主流。

数位声音讯号的品质，可以先从其取样频率以及解析度看出。所谓取样频率，就是指要将一秒钟的声波分割成几个点来加以数字化，也就是一秒钟要取样几次：如 CD 音质的取样频率是 44.1kHz，而现今录音工业，甚至 DVD 的标准则是 96 kHz。另外，解析度是取样值要以几位元的数字来表示：CD 音质是 16 位元，而 DVD 的标准则是 24 位元。取样频率与解析度越大，则数位声讯的品质就越细腻。

Analog Audio Storage「 类比声音储存 」

指的是以磁带储存声波引起的电压变化，像是一般的卡式录音座用的录音带。

Analog Modeling「 类比模拟 」

类比模拟，是以 Roland 组合物件声音模型 (COSM) 技术为基础的声音合成型态。COSM 技术是利用复杂的软件演算，以模拟出各种器材的声音特性 —— 而所谓的类比模拟，就是用来模拟类比振荡器 (Analog Oscillator)。

类比模拟使用先进的数位讯号处理 (DSP) 技术，重现出传统类比合成器 (Analog Synthesizer) 的标准波形 (像是锯齿波、方波、三角波…等等)。除此之外，类比模拟还可以产生出传统类比线路所无法产生的波形，像是超级锯齿 (Super Saw) 波与回授振荡器。类比模拟提供了类比合成器特有的平滑音质，并加入了类比合成器所没有的先进数位控制，像是稳定音调、音色记忆、效果处理、以及 MIDI 控制。

Roland 的超级锯齿波是一种类比模拟，它让七个可各自微调的锯齿波合在一起，并以单一发声数来发出肥厚的音色。而回授振荡器是模拟吉他前级放大器的特殊声波，可做为强而有力的前导音色。

Analog Oscillator 「 类比振荡器 」

可自行产生振荡讯号，以发出各种标淮波形 (如正弦波、锯齿波、方波、三角波等) 的电子元件。

Analog Synthesizer「类比合成器 」

这是最早期的合成乐器 (Synthesizer) ，它利用类比振荡器 (Analog Oscillator) 产生的波形，再加以合成处理，以产生出各种不同的音色。虽然它的音色没有办法完全模拟真实乐器，但其特殊的音质表现，到现在仍然广受使用，尤其是在电子舞曲界。

Antifeedback Processor「 回授抑制处理器 」

这种处理器能迅速扫描、侦测、并将发生回授的频率点压低，以避免回授的发生。这种器材在自动侦测模式中，会慢慢地提高音量，以精确地压下回授频率，并将确保音质的减损到最小。

这里的回授 (Feedback) ，指的是当麦克风与喇叭处于某一距离或音量过大时，麦克风又将喇叭的声音收入，造成极为刺耳的尖锐杂音。

Arming Tracks「 待录音轨 」

准备要录音的音轨。也称做「Record Ready」(录音待命)。

Arpeggiator「 自动琶音器、自动琶音功能 」

这是 MIDI 编曲器 (Sequencer) 的一种功能，您只需压下琴键或按键，就可以照着预定的型态，产生出一串旋律或节奏。

早期内建有自动琶音功能的合成器 (Synthesizer) 中，其琶音功能是以所弹的音符产生出节奏的变化。一般的应用方式就是弹下一个和弦并压住，让这些和弦音产生上行、下行、或随机的琶音。您可以决定琶音的速度。

现在的自动琶音功能，像是内建在XP系列合成器里的，则提供更独特、更具风格的琶音样式，一些用键盘很难演奏出来的技巧，例如吉他刷弦、竖琴的琶音，都可以利用自动琶音功能来帮您产生出来，提升您在演奏上的可能性。

Arranger Keyboard「 自动伴奏键盘 」

这是一种键盘乐器，只要一个人就可以弹奏出乐团般的合奏音乐。

Attack「 起音 」

用在音色上，是指声音在发声瞬间的特性。例如所谓 Attack 较强的乐器音色，表示在一弹奏该乐器的瞬间，音量会较快冲到最高点，例如钢琴、铁琴。而 Attack 较弱的乐器，则是弹奏时，会较慢到达最高音量，例如弦乐器。

在音源机、合成器当中，通常会提供可调整 Attack 特性的参数，一般常见者为 Attack Time (起音时间)。若 Attack Time 的值越大，表示该音色从一弹奏 (压下琴键) 到音量最大之间的时间会较长，声音听起来就好像慢慢起来的感觉；而 Attack 值越小，表示从弹奏 (压下琴键) 到音量最大之间的时间会较短，声音听起来就比较铿锵。

另外有的合成器 (Synthesizer) 也提供 Attack Level (起音音量) 的调整参数。这是用来决定弹奏该音色所冲到的最高音量是多大。

Audio DAT 「 DAT 」

这是一种记录声音的磁带，并且是以数位的格式储存两轨。它通常是用在前期母带储存， 而 DAT 录音座也多具有数位立体声输入。

Automated Mix「 自动混音记录 」

混音器上的推杆动作、左右相位、以及其他混音动作的记录，您可以将这些记录播放出来，以让最后的混音更为精准。

Automation「自动混音 」

对于混音器设定变化的记录及播放。

为什么要有自动混音的功能？简单来说，因为人只有两只手，但是录音很少不超过两个音轨的，这么多的音轨都需要进行混音调整，而且多半是同时进行。如果有了自动混音功能的帮助，您就可以分别将各音轨的混音动作给记忆下来，最后再一起播放出来，您甚至还能够进行自动混音的编修，让您的混音更精确。

在 VS 系列录音座上的自动混音功能，还能够让您记录同一轨中的效果器变换，如此，您就可以很方便地重覆使用它内建的效果器。例如说，您在同一轨前后录进了前导 (lead) 吉他与节奏吉他，您就可以用自动混音功能，在节奏吉他时使用较温和的 VS 吉他多重效果，而到前导吉他时，自动切换到强烈的 COSM 破音效果。

Aux「 辅助、辅助信号线路／端子 」

这是「Auxiliary (辅助)」的缩写，它是一种额外的讯号线路设计。在一般的音响器材上，除了正式的输出与输入端口之外，常常还会配备几个标有「Aux」的输出入端口，做为预备用的接续端。当您有特别的应用，例如要做额外的声音输出或输入时，就可以利用这种端口。这种预备端口或线路，不论输出入，我们统称为 Aux 。

以一台混音机为例，它会将输入的声音讯号加以合并，然后再一起从主输出声道送出；然而混音机的输出线路一般不会只有主输出声道，为了更弹性的应用，例如希望声音能经过外接效果器的处理，或是除了主喇叭之外，还需要把声音送到另外的监听喇叭，因此混音机通常还会预备有 Aux 输出端口。

另外以家用音响为例：其前级扩大机通常已具备有几个输入端口，且已经标示要接自哪一台机器 —— 如 CD、Tape、Tuner 等等，然而在这些输入端口中，通常还会再配备 1 到 2 对的 Aux 输入端口，例如当您想要接用两台 CD 时，其中一台可以直接接到标有「CD」的输入端，另一台则可以利用其中一对 Aux 端口。

在混音机中，就需要有另外的线路，以将声音讯号导至这些额外的输出端口，这样的线路我们可以称之为 Aux ；或是这些额外的输出端口我们也简称为 Aux 。

透过 Aux 线路，您可以规划出更复杂的 PA 或录音系统，例如除了主喇叭之外，您还可以只传送某几轨的声音到特定的监听喇叭。

B

Back Up「 备份 」

将您所记录或编辑的资料，另外再储存一份到可抽换的储存媒体。利用备份，您可以在器材本体上腾出较多的储存空间，或是做为副本资料以避免资料意外损毁。最快速的备份方式是利用 SCSI 端口将资料储存到可更换的储存周边。又如 DAT 、光碟刻录机也是可备份数位资料的储存装置。

Balanced「 平衡式 」

这是一种声讯传输导线的型态，采用三个脚位 (即三芯导线) ，利用相位抵消的原理，可以增强实质讯号并抑制杂讯。在应用上，较长的距离会使用平衡型的接线。

Band「 波段、频带 」

在等化器「EQ」中，它並不是「乐团」的意思，而是指一段频率范围。例如 Middle Band ，就是中频波段的意思。

Band-Pass Filter「 带通滤波器 」

这是滤波器的一种，它只留下Cutoff Frequency(截频点)左右某音频范围的组成泛音，而将其他的音频去掉。

Bandwidth「 频宽 」

在等化器「EQ」中，它指的是一个波段「Band」的宽度。当您对等化器中某个中央频率点 (Central Frequency) 进行放大或压低的调整时，所影响的并非只有该频率点上的声音，而是该波段频宽中的所有声音。例如，假设等化器上每一个波段的频宽为 1 个八度音，则当您对等化器上加强 100 Hz 的声音时，实际上所加强的会是 75Hz-150Hz 之间的声音。

Bank「 群组 」

在音源机或合成器中，有时会将音色分成几个组，例如 XV-5080 中的预设音色分成 A-H 组，每一组中各自有 128 种音色 —— 一个组就叫做一个 Bank 。

在综合效果器中，也会将效果分成几个 Bank ，但通常是 4、5 个效果为一组。

为什么要将音色或效果再分成数个群组？原因很简单，就是让音色选择的操作更方便一些；另外一个原因，则是避免 MIDI 讯号的限制。

Bank Select「 群组选择 」

在MIDI的讯号中是以一个位元组 (Byte) 为单位，而一个位元组能表达 0-127 的数字。 MIDI 是以 Program Change 这种一个位元组的讯息来指定音色的，也就是说，这个讯息只能指定 128 种音色，因为一个位元组能表达的数字只能 0-127。

然而目前的音源机、合成器所内建的音色，绝对都已远超过 128 种，光靠 Program Change 这种讯息，不足以选用到所有的音色，因此，还要再加入 Bank Select ，也就是所谓的群组选择讯息。

例如，XV-5080 中的预设音色分成 A-H 个 bank ，一个 Bank 中各有 128 种音色，因此，当您要从编曲机或电脑指定其某一组中的某一音色时，就要先给该组所对应的Bank Select 讯息，然后再给 Program Change 讯息。如想要指定 A-054 号的音色，就要先送 A 组的群组选择码，然后再送 Program Change 54 号的讯号给 XV-5080。

Bank Select 讯息是由两个位元组所结合而成，可以表达 0-16383 的数字；再加上 Program Change，则总共可以表达 2 百多万个数字，换句话说，未来的音源器、合成器就算内建再多的音色，也都可以透过 Bank Select 与 Program Change 讯息加以指定。

Beat Loop「 循环节奏、节奏 Loop 」

一段节奏乐句的取样音色，在播放时会不断地反复。

您可能在合成器中会发现某些音色，当您弹下琴键时会播放出一段节奏乐句，并且不断反复直到您放开琴键为止，这就是 Beat Loop。另外，您也可能会利用取样机对一段节奏音乐做取样，并在播放时让它不断反复，这也是 Beat Loop。

Bit「 位元 」

在数位的领域中，所有的资料都是被数字化，并且是以 0 与 1 的组合来代表各个数目，也就是数学上所谓的二进位制。

一个 0 或 1 ，就是一个位元。例如当我们说一个 2-位元 的数字，就是指它是一个两位数的二进制数字，有 00、01、10、11 这 4 种组合，分别代表 0、1、2、3 的数字。

一个 3 位元的数字，能代表 0-7 这 8 个数字；而一个 4 位元的数字，能代表 0-15 这 16 个数字。以此类推下去，一个 16 位元的数字，能代表 0-65535 的数字。简单地说，位元数越大，所能代表的数字范围就越广。

Bouncing Tracks「 并轨 」

将几个音轨合并成为一个单音音轨或一个立体声音轨。通常这个动作会将原有的录音轨给洗掉，但若是利用虚拟轨「Virtual Track」，您就可以将原来的音轨保留下来，日后再做其他不同的混音尝试。

BPM「 节拍速度 」

BPM是「Beats Per Minute」(每分钟几拍) 的缩写，用来量度一首音乐的速度。例如若一首音乐的 BPM 是 120，则表示它的速度是一分钟 120 拍。

BPM Calculation and Sync「 BPM计算与同步 」

这种功能是用来计算一首歌曲的速度「BPM」，并加以显示。它的原理是抓取歌曲中大鼓出现时的较强音量，以分析出每分钟有几拍。您也可以透过 BPM 调整或 Tap Tempo 按键，来计算出 BPM，基本上只要一算出 BPM 之后，就可以进行效果同步以及 MIDI 时序码的传送，并且也可以很方便地让各种不同来源的音乐加以速度同步，像是来自唱盘、CD、以及任何可以跟 MIDI 同步讯息相容的 MIDI 器材的音乐。
